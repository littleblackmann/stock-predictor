### 2. 使用者資料搬遷至 AppData（核心改動）
**目的**：將程式檔案與使用者資料徹底分離，更新程式時不影響使用者的設定、模型、記錄。

#### 新增模組：`data/data_paths.py`
統一管理所有資料路徑，所有使用者資料存放在：
```
C:\Users\{使用者}\AppData\Local\台股預測分析系統\
├── config.json              ← API Key、設定
├── watchlist.json            ← 自選股
├── prediction_log.csv        ← 預測記錄
├── auto_retrain_cooldown.json ← 重訓冷卻期
├── stock_list_cache.json     ← 股票清單快取
├── tw_holiday_cache.json     ← 假日快取
├── models/saved/             ← 訓練好的模型
├── cache/chip/               ← 籌碼資料快取
└── logs/                     ← 應用程式日誌
```

### 3. 自動更新模組

#### 新增模組：`updater/auto_updater.py`
| 功能 | 說明 |
|------|------|
| `check_for_update()` | 透過 GitHub Releases API 檢查最新版本 |
| `download_and_apply()` | 下載 ZIP → 寫更新批次腳本 → 退出 → 腳本覆蓋程式 → 重啟 |
| `skip_version()` | 使用者可跳過特定版本 |
| 版本比對 | 語意化版本號比較（1.2.3 格式） |

#### 更新流程
```
啟動後 5 秒 → 背景檢查 GitHub Releases
    ↓ 有新版本
彈出對話框：「立即更新 / 跳過此版本 / 稍後提醒」
    ↓ 立即更新
下載 ZIP → 進度條 → 寫 bat 腳本 → 退出程式
    ↓
bat 腳本等 2 秒 → xcopy 覆蓋程式 → 重啟 exe → 清理暫存
```

#### UI 整合（`ui/main_window.py`）
- 啟動後 5 秒背景執行 `_check_for_update_bg()`
- 發現新版本 → `_show_update_dialog()` 三選一對話框
- 「立即更新」→ `_do_update()` 下載 + 進度條 + 重啟


## 架構變更摘要

### Before（v1.0.0）
```
台股預測分析系統/
├── 台股預測分析系統.exe    ← 程式
├── config.json            ← 資料（混在一起）
├── watchlist.json          ← 資料
├── models/saved/           ← 資料
└── ... DLL                ← 程式
```
更新 = 整包替換 = 資料不見

### After（v1.1.0）
```
C:\Users\{user}\AppData\Local\台股預測分析系統\   ← 使用者資料（永久）
├── config.json, watchlist.json, models/, cache/, logs/

E:\台股預測分析系統\                              ← 純程式（可替換）
├── 台股預測分析系統.exe, version.json, DLLs...
```
更新 = 只換程式 = 資料不受影響

## ⚠ 給後續開發者的重要注意事項

### 路徑相關
1. **所有新增的使用者資料檔案，必須在 `data/data_paths.py` 新增路徑常數**
2. **絕對不要在程式碼中用 `os.path.dirname(__file__)` 來定位使用者資料**，要用 `data_paths` 模組
3. **打包時（.spec）不需要包含使用者資料**，程式首次啟動時會自動在 AppData 建立

### 更新相關
4. **更新版本時記得修改 `version.json`**
5. GitHub 倉庫設定在 `config.json` 的 `github_owner` 和 `github_repo` 欄位
6. 上傳 Release 時需附上 ZIP 格式的 asset（build.py 產出的 ZIP）

### 遷移相關
7. **遷移只執行一次**（AppData 下有 `.migrated` 標記檔）
8. 若需要重新遷移，刪除 `%LOCALAPPDATA%/台股預測分析系統/.migrated` 即可

## 未來優化（等功能穩定後再做）

### 打包加速 / 差量更新
目前每次更新都需要完整打包（5~15 分鐘）+ 上傳 646MB ZIP，非常耗時。
等 bug 修完、功能穩定後，可以考慮：
1. **差量更新**：只下載有變動的檔案（幾 MB），不用每次下載整包 646MB
2. **程式碼與引擎分離**：TensorFlow/PySide6 等重型 DLL 作為引擎包，Python 程式碼單獨打包，更新時只替換程式碼部分
3. **增量打包**：`build.py` 不刪除 dist，讓 PyInstaller 用快取加速
