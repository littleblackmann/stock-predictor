## Day 4 補充（自動重訓系統）

### 核心設計：全自動準確率驅動重訓
使用者目標：每天持續預測，讓模型隨時間越來越準確，不需要手動介入。

**觸發條件（三個同時滿足才重訓）：**
- 最近 20 筆 actual 記錄中，準確率 < 52%
- 距上次自動重訓超過 3 天（冷卻期）
- 上述條件在 backfill 完成後自動判斷，使用者完全不需要操作

**業界標準參考：**
- 股票方向預測能穩定超過 55% 已屬有效；52% 設為門檻（略高於隨機），確保提早抓到模型偏掉的趨勢

**新增 `data/prediction_logger.py`**
- `check_auto_retrain_candidates(threshold=0.52, min_records=20, cooldown_days=3)`：掃描所有有記錄的股票，回傳符合重訓條件的清單
- `mark_retrained(symbol)`：記錄重訓時間至 `auto_retrain_cooldown.json`
- `_load_cooldown_log()`：讀取冷卻記錄

**新增 `workers/auto_retrain_worker.py`**
- `AutoRetrainWorker(symbols)`：對多支股票依序執行背景靜默重訓
- 流程：下載 700 天資料 → 特徵工程 → 重訓 LSTM → 批次萃取特徵 → 重訓 LightGBM
- `symbol_done = Signal(str, bool)`：每支完成後通知
- `all_done = Signal(int, int)`：全部完成後通知

**修改 `ui/main_window.py`**
- `_backfill_in_background`：backfill 完成後延遲 1 秒觸發 `_check_auto_retrain()`
- `_check_auto_retrain()`：呼叫候選判斷，有候選則啟動 AutoRetrainWorker
- `_on_auto_retrain_symbol_done()`：每支完成後呼叫 `mark_retrained()`，狀態列顯示「✅ 0050 模型已自動更新」
- `_on_auto_retrain_all_done()`：全部完成後顯示彙總訊息

**App 啟動全自動流程：**
```
開啟 App → backfill 回填 → 自動檢查準確率 → 背景靜默重訓 → 狀態列通知
```
