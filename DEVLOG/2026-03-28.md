## 完成項目

### 1. 更新機制修正（v1.2.4）

#### 問題：更新後程式未重啟、檔案未覆蓋
- **根因**：bat 腳本只等 2 秒就 xcopy，但 exe 還在執行中，Windows 鎖住檔案 → xcopy 靜默失敗
- **修正**：
  - bat 腳本改用 `tasklist` 輪詢等待進程退出（最多 15 秒）
  - 超時後 `taskkill /F /PID` 強制結束
  - 確認程式死了才開始 xcopy
  - xcopy 失敗時顯示錯誤訊息（不再靜默）
- **影響範圍**：`updater/auto_updater.py`

#### 問題：版本號顯示 v0.0.0
- **根因**：PyInstaller 把 `version.json` 打包在 `_internal/`（即 `_MEIPASS`），但程式只找 exe 旁邊的路徑
- **修正**：`get_current_version()` 加入 `_MEIPASS` fallback 路徑
- **影響範圍**：`updater/auto_updater.py`

#### 問題：設定頁「檢查更新」無法直接觸發更新
- **根因**：按鈕只顯示「請關閉設定視窗」文字，沒有實際動作
- **修正**：發現新版本後自動關閉設定視窗，直接呼叫主視窗 `_do_update()`
- **影響範圍**：`ui/settings_dialog.py`

#### 進度對話框
- 更新時顯示 420x120 的 `QProgressDialog`
- 顯示下載百分比 + MB 數（如 `123.4 MB / 646.0 MB (19%)`）
- `ApplicationModal` 模式，使用者無法操作主視窗

### 2. 更新日誌精簡化（v1.2.5）
- 「關於/更新」分頁只顯示最新一筆 changelog
- 減少視覺干擾，使用者只需看到最新版本的改動

### 3. 差量更新機制（v1.2.6）

#### 原理
每次打包後，掃描 dist 內所有檔案計算 SHA256 → 存為 `build_manifest.json`。
下次打包時，比對新舊 manifest → 只把有變動的檔案打成 `patch.zip`。

#### build.py 改動
| 新增功能 | 說明 |
|---------|------|
| `_hash_file()` | 計算檔案 SHA256 |
| `generate_manifest()` | 掃描 dist 資料夾產生 `{相對路徑: hash}` |
| `create_patch_zip()` | 比對新舊 manifest，產生差量包 |
| `build_manifest.json` | 儲存在專案根目錄，供下次打包比對 |
| `manifest.json` | 放入 dist 內，供未來本地比對 |

#### 打包流程變化
```
Before（3 步驟）：
  環境確認 → PyInstaller → 壓縮 full.zip

After（4 步驟）：
  環境確認 → PyInstaller → 產生 manifest + patch.zip → 壓縮 full.zip
```

#### auto_updater.py 改動
- `check_for_update()` 同時尋找 `patch_*.zip` 和 full zip asset
- 優先使用 patch（小檔案），找不到才 fallback 到 full
- 回傳值新增 `is_patch`、`full_url` 欄位
- `download_and_apply()` 新增 `full_url` 和 `is_patch` 參數

#### 上傳 Release 流程
```bash
# full + patch 都上傳
gh release create vX.X.X \
  "台股預測分析系統_vX.X.X.zip" \
  "台股預測分析系統_vX.X.X_patch.zip" \
  --title "vX.X.X — 更新標題" \
  --notes "更新說明"
```

#### 效益
| 更新類型 | 下載量 | 時間 |
|---------|--------|------|
| 完整更新 | ~646 MB | 數分鐘 |
| 差量更新 | ~3-10 MB | 幾秒 |
