## 完成項目

### 1. 預測記錄無法回填實際結果（v1.4.3）

#### 問題
預測記錄的「實際」「漲跌%」「正確」三個欄位永遠顯示「—」，按「更新實際結果」無反應（顯示「已是最新」）。

#### 根因：utf-8-sig + append 模式 = BOM 汙染

`prediction_logger.py` 的 `append()` 使用 `encoding="utf-8-sig"` 在 append 模式寫入 CSV。Python 的 utf-8-sig 在每次開檔追加時都會在資料前插入 3 bytes 的 BOM（`\xef\xbb\xbf`）：

```
第 1 次寫入（建檔）：BOM + header + row1  → 正常
第 2 次寫入（追加）：BOM + row2           → prediction_date 被 BOM 汙染
第 3 次寫入（追加）：BOM + row3           → 同上
```

回填時 `date.fromisoformat("\ufeff2026-04-01")` 拋出 ValueError，被 `except Exception: continue` 靜默吞掉。所以第 2 筆以後的記錄永遠無法回填，且完全看不到錯誤訊息。

#### 修復（3 處）

| 位置 | 修復 |
|------|------|
| `append()` | `utf-8-sig` → `utf-8`，加上空檔案檢查 |
| `load_all()` | 讀取後清除舊資料中殘留的 BOM 字元（`\ufeff`），向後相容 |
| `_save_all()` | `utf-8-sig` → `utf-8` |
| `backfill` except | 改為 `logger.warning` 不再靜默吞錯 |

**影響範圍**：`data/prediction_logger.py`

### 2. 自動更新重啟循環（v1.4.3）

#### 問題
上傳 v1.4.3 後，使用者桌面的 exe（v1.4.2）偵測到新版，開始自動更新。更新完重啟後又偵測到「有新版」，導致 app 重啟 6-7 次才穩定。

#### 根因：bat 腳本中文路徑編碼問題

更新用的 bat 腳本由 Python 以 UTF-8 寫入，但 cmd.exe 用系統 codepage（cp950）讀取。bat 中用 `echo` 寫 version.json 到中文路徑（`台股預測分析系統`）不可靠：

```bat
REM 這行在 cp950 下可能失敗
>"C:\...\台股預測分析系統\version.json" (
    echo {"version": "1.4.3"}
)
```

version.json 寫入失敗 → 重啟後版本號仍是 1.4.2 → 又偵測到新版 → 無限循環。

#### 修復

改為在 Python 端預寫 `version.json` 到解壓目錄，xcopy 一起複製過去，完全不依賴 bat 寫中文路徑：

```python
# 在 Python 端寫好，xcopy 會一起複製
ver_in_source = os.path.join(source_dir, "version.json")
with open(ver_in_source, "w", encoding="utf-8") as vf:
    json.dump({"version": new_version}, vf)
```

同時移除 bat 中的 `echo` 中文訊息，避免編碼問題。

**影響範圍**：`updater/auto_updater.py`

## 技術筆記：預測記錄與模型的關係

預測記錄**不直接參與模型訓練**，而是作為「觸發器」：

```
啟動 → 回填實際結果 → 計算每支股票最近 20 筆準確率
→ 低於 55% 的股票自動觸發重訓（用最新 2500 天歷史資料重新訓練）
→ 同一股票 3 天冷卻期
```

目前每支股票只有 4-6 筆記錄，尚未達到 20 筆觸發門檻。預估再使用 2-3 週後開始自動監控。

### 5. 成交量異常偵測（v1.5.0）

在 `features/feature_engineer.py` 新增 4 個量能特徵：

| 特徵 | 說明 |
|------|------|
| `vol_anomaly` | 成交量 Z-score（(當日量 - MA20) / STD20），偵測統計上的異常量 |
| `vol_breakout` | 爆量突破旗標（當日量 > 2 倍 MA20 時為 1） |
| `vol_price_diverge` | 量價背離（放量不動 = 1，縮量大動 = -1，正常 = 0） |
| `vol_trend` | 量能趨勢（vol_ma5 / vol_ma20），>1 表示近期量能放大 |

已同步更新 `get_feature_cols()` 和 `get_transformer_input_cols()`，Transformer + LightGBM 都會使用這 4 維新特徵。

**影響範圍**：`features/feature_engineer.py`

### 6. 分市場狀態訓練（v1.5.0）

在 `models/lgbm_classifier.py` 新增行情專用模型訓練與混合預測：

#### 訓練階段
- 根據 `market_regime`（1=多頭, -1=空頭, 0=盤整）將資料分群
- 每群資料 ≥ 200 筆時訓練獨立的 LightGBM（200 estimators, 15 leaves）
- 訓練完自動儲存（`{symbol}_regime_{bull/bear/side}_model.pkl`）

#### 預測階段
- `predict()` 新增 `current_regime` 參數
- 若有對應行情模型：`最終機率 = 主模型 × 65% + 行情模型 × 35%`
- 若無對應模型：退回主模型 100%

#### 整合
- `prediction_worker.py` 在呼叫 `predict()` 前取出 `latest_regime`，傳入 `current_regime`
- SHAP 特徵標籤新增 4 個量能中文名稱

**影響範圍**：`models/lgbm_classifier.py`、`workers/prediction_worker.py`
