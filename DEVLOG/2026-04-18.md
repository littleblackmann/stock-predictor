# Day 14 — 2026-04-18

## 主題：準確率調校（情緒降權 + 類別平衡 + 美股 NaN 修復）

> 使用者回報：父母使用近一個月，預測準確率不理想。分析 `prediction_log.csv` 35 筆有效記錄後發現系統性偏見。

---

## 診斷：準確率統計

| 指標 | 數字 |
|------|------|
| 整體準確率 | **45.7%**（16/35）— 低於 50% |
| 預測「漲」命中率 | **11%**（2/19）— 反指標等級 |
| 預測「跌」命中率 | **88%**（14/16） |
| 實際市場漲/跌 | 4 漲 / 31 跌（89% 在跌） |

這段期間市場 89% 在跌（明顯空頭），但模型 54% 預測漲 → **過度看多**。
若單純「全部預測跌」反而有 89% 準確率，比模型好。

### 根因分析

1. **GPT 情緒權重過高**（最大 ±7.5%）：很多 `up_prob` 落在 0.45~0.52 的臨界區間，情緒分數能把模型從「跌」翻成「漲」。
2. **美股 `^GSPC`/`^SOX`/`^VIX` 下載未啟用 `repair=True`**：Day 13 修了台股的 NaN 但漏了美股，造成 4 個美股隔夜特徵偶發 NaN。
3. **LightGBM 無類別平衡**：台股歷史上漲比率略高（~52%），但極端行情期間模型仍然傾向多頭。

---

## 完成項目

### 1. 降低 GPT 情緒權重（立即生效，不用重訓）

- `SENTIMENT_WEIGHT` 基礎值 15% → 8%（上限同步）
- 最大情緒調整量：±7.5% → ±4%
- 臨界預測（如 0.48）不再被情緒 flip

**影響範圍**：`workers/prediction_worker.py`

### 2. LightGBM 啟用類別平衡（需要重訓才生效）

```python
LGBM_PARAMS = {
    ...
    "is_unbalance": True,
    ...
}
```

**影響範圍**：`models/lgbm_classifier.py`

### 3. 預測記錄新增 `raw_up_prob` 欄位

未加 GPT 情緒的原始模型機率一併記錄，未來可分析情緒融合是否真的有幫助。

**影響範圍**：`data/prediction_logger.py`

### 4. 美股資料啟用 `repair=True`

```python
df = ticker.history(period=f"{period_days}d", auto_adjust=True, repair=True)
df = df.dropna(subset=["Close"])
```

**影響範圍**：`workers/prediction_worker.py` → `_download_us_market_data()`

## 誠實免責

1. 35 筆樣本統計雜訊大，89% 空頭屬極端行情
2. 若市場轉為盤整/多頭，這些調整可能使預測偏空
3. 日線二元預測的模型理論天花板約 52~58%
4. 真正想要高命中，應該「只看 high confidence 那批」或「拉長預測週期」
