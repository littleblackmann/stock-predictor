## 完成修正

### Bug Fix 1：模型未按股票代號分開存（重要）
**問題**：切換股票時會載入其他股票訓練的模型，導致預測錯誤
**修正**：
- `LSTMExtractor(symbol=...)` 和 `LGBMClassifier(symbol=...)` 加入 symbol 參數
- 模型檔名改為 symbol 相關命名，例如：
  - `lgbm_0050_TW.pkl` / `lgbm_0050_TW_scaler.pkl` / `lgbm_0050_TW_timestamp.json`
  - `lstm_0050_TW.keras` / `lstm_0050_TW_scaler.pkl`
- `needs_retrain(symbol)` 改為按 symbol 各自檢查
- `main_window._update_model_status()` 改用當前輸入框的 symbol

### Bug Fix 2：LSTM 特徵萃取效能問題
**問題**：`_extract_all_lstm_features()` 用 for loop 每筆呼叫一次 `predict()`，700天資料 = 700次推論，非常慢
**修正**：
- 用 `np.stack` 一次組出所有滑動窗口 → shape: `(n_windows, 60, n_features)`
- 單次 `predict(batch_size=256)` 批次推論
- 速度預估提升 20~50 倍

### Bug Fix 3：進度條數字倒序
**問題**：步驟 8（3日走勢）emit 96%，步驟 9（整理圖表）emit 95%，順序錯誤
**修正**：步驟 8 = 整理圖表（95%），步驟 9 = GPT 3日走勢（96%）

## 修改的檔案
- `models/lgbm_classifier.py` — symbol 分開存、needs_retrain 加 symbol 參數
- `models/lstm_extractor.py` — symbol 分開存
- `workers/prediction_worker.py` — 傳入 symbol、批次萃取、修進度順序
- `ui/main_window.py` — _update_model_status 傳入當前 symbol
