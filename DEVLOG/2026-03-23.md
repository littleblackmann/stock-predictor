### 打包版二度修復（Day 5 深夜 #2）

**問題：** 第一次 bug fix 後重新打包，打包版預測仍然失敗（同樣的 `'NoneType' object is not subscriptable`）。

**診斷：** 讀取打包版 log 發現 `period="2y"` fallback 也失敗。問題不是 timezone cache miss，而是 `curl_cffi` 在 PyInstaller 打包環境中無法發送 HTTPS 請求。yfinance 優先使用 `curl_cffi` 取得 cookie/crumb → HTTPS 請求失敗 → crumb = None → API 回傳 None → TypeError。

**修復：**
- `main.py` 最頂端加入 `curl_cffi` 封鎖（僅在 `sys.frozen=True` 時生效）
- 強制 yfinance 改用 `requests`（已正確打包）取得 cookie/crumb
```python
if getattr(sys, 'frozen', False):
    sys.modules.setdefault('curl_cffi', None)
    sys.modules.setdefault('curl_cffi.requests', None)
```

**第二次重打包結果：** `台股預測分析系統_v1.0.0.zip`（638 MB）

**測試結果：** 打包版 log 顯示啟動 + 假日抓取正常（24 筆），但之後沒有任何預測相關 log。AppLoader 背景執行緒靜默 crash，主視窗永遠不出現。

### 打包版三度修復 — curl_cffi shim（Day 5 深夜 #3，最終修復）

**根因深入分析：**
- 第一次封鎖用 `sys.modules['curl_cffi'] = None` → 讓 `import curl_cffi` 拋 ImportError
- 但 yfinance 1.2.0 的 **8 個檔案**在模組頂層直接 `from curl_cffi import requests`，**全部沒有 try/except**
- curl_cffi 是 yfinance 的**硬依賴**，不是可選依賴
- 封鎖 curl_cffi → yfinance 無法 import → AppLoader 執行緒 crash（無 try/except → 靜默死亡）

**解決方案：curl_cffi Shim（替身模組）**
- 不封鎖 curl_cffi，而是用標準 `requests` 庫模擬 curl_cffi 的完整 API
- 在 `main.py` 中 inline 建立假的 curl_cffi 模組階層（`types.ModuleType`）
- 安裝到 `sys.modules` 後，yfinance 的 `from curl_cffi import requests` 會拿到我們的 shim
- 僅在 `sys.frozen=True`（打包環境）時啟動，VSCode 原始碼不受影響

**Shim 模擬的 API：**
- `curl_cffi.requests.Session(impersonate=...)` → 標準 `requests.Session` + Chrome User-Agent
- `curl_cffi.requests.Response` → 標準 `requests.Response`
- `curl_cffi.requests.session.Session` → 同上（isinstance 檢查用）
- `curl_cffi.requests.exceptions.*` → 對應的標準 `requests.exceptions` 類別
- `curl_cffi.requests.exceptions.DNSError` → 自定義（繼承 ConnectionError）
- `session.cookies.jar` → 加上 `.jar` 屬性指向自身（curl_cffi 相容）

**Chrome User-Agent 的必要性：**
- curl_cffi 的 `impersonate="chrome"` 會偽裝 TLS 指紋和 HTTP 標頭
- 不加 User-Agent → Yahoo Finance 把請求當機器人 → rate limit → crumb 取得失敗
- 加上 Chrome-like User-Agent 後，標準 requests 也能正常取得 cookie/crumb

**其他改進：**
- AppLoader.run() 加入 try/except + logger.critical，避免背景執行緒靜默 crash
- build.py 移除 emoji 字元（cp950 編碼不支援 ✅❌📌 等 emoji）

## Day 6 預告：模型自我進化機制

**目標：** 讓預測系統像 AlphaGo 一樣「越用越準」，而不是停在 50-55% 震盪。

**目前瓶頸分析：**
1. 訓練資料固定 700 天滑動窗口 → 資料量不增長
2. 特徵組合固定（13/20 維）→ 不會發現新有效特徵
3. 52% 準確率門檻太低 → 幾乎等於猜硬幣
4. 每 7 天重訓 = 從頭訓練 → 不保留歷史學習成果

**預計改動方向：**
- 累積式訓練窗口（資料越用越多）
- 特徵自動篩選（淘汰無效特徵、發現新組合）
- 多模型 ensemble（不同時期模型投票）
- Online learning / incremental training（微調而非重訓）
- 提高準確率門檻（55%+）
