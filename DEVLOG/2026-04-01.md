## 完成項目

### 1. K 線圖日期缺少當天資料（v1.4.1）

#### 問題
K 線圖最後一根 K 棒停在 2026-03-27（週五），但當天是 2026-03-30（週一），22:50 已收盤超過 9 小時，應有當天資料。

#### 根因
`yfinance_adapter.py` 的 `fetch_history()` 中 `end_date = datetime.today()`，但 yfinance 的 `end` 參數是 **exclusive**（不含當天），所以 `end="2026-03-30"` 只拿到 2026-03-29 以前的資料。

#### 修復
```python
end_date = datetime.today() + timedelta(days=1)  # +1 讓 yfinance 包含今天
```

**影響範圍**：`data/yfinance_adapter.py`

### 2. 啟動偶發閃退（v1.4.1）

#### 問題
系統有時候剛打開就直接被關掉，沒有任何錯誤訊息。

#### 根因（3 個漏洞）

| 漏洞 | 位置 | 說明 |
|------|------|------|
| 啟動序列無保護 | `main.py:128-130` | `migrate_from_old_location()` 和 `cleanup_legacy_models()` 在 QApplication 建立前執行，無 try/except。檔案被鎖 → crash，無 log |
| AppLoader done signal 未發射 | `main.py:115-121` | exception handler 不 emit `done` → 主視窗永遠不建立 → splash 永遠卡住 |
| 模型清理單檔連鎖失敗 | `data_paths.py:134,138` | `os.remove()` 無 try/except，一檔鎖定整個函式 crash |

#### 修復

| 位置 | 修復 |
|------|------|
| `main.py:128-130` | 包在 try/except，失敗不阻擋啟動 |
| `main.py:121` | 補上 `self.sig.done.emit()` |
| `data/data_paths.py:134,138` | 每個 `os.remove()` 獨立 try/except |

**影響範圍**：`main.py`、`data/data_paths.py`

### 4. 預測記錄漲跌%顯示 0.00% / nan%（v1.4.2）

#### 問題
預測記錄的「漲跌%」欄位全部顯示 0.00% 或 nan%，沒有正確的漲跌幅。

#### 根因（3 個 bug）

**Bug 1（0.00%）**：`prediction_logger.py:127` — `pred_close` 用 `"after"` 方向找收盤價。假日（如週日）預測時，pred_close 和 target_close 都往後找到同一個交易日（週一）→ 差額 = 0。
```
週日預測 → pred_close = 週一收盤, target_close = 週一收盤 → 0%
```

**Bug 2（nan%）**：yfinance 偶發回傳 NaN Close（如 03-31），`_near_price()` 沒有過濾 NaN → 計算出 nan → 寫入 CSV → 顯示 "nan%"。

**Bug 3（舊記錄不重算）**：回填邏輯只處理 `actual` 為空的記錄（line 86），已經被錯誤回填的記錄（actual_return = "0.00" 或 "nan"）不會被修正。

#### 修復

| Bug | 修復 |
|-----|------|
| 0.00% | `pred_close` 方向從 `"after"` 改為 `"before"`（往前找最近交易日收盤價） |
| nan% | `_near_price()` 加 `math.isnan()` 檢查，跳過 NaN 值 |
| 舊記錄 | pending 條件加上 `actual_return in ("", "0.00", "nan")`，錯誤記錄自動重算 |

#### 驗證
```
修復前：pred_close=73.9（週一）, target=73.9（週一）→ 0.00%
修復後：pred_close=75.0（週五）, target=73.9（週一）→ -1.47%
```

**影響範圍**：`data/prediction_logger.py`
