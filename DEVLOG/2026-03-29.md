## 完成項目

### 1. 籌碼面二階衍生特徵（6 個新特徵）

在原有 7 個籌碼衍生特徵基礎上，新增 6 個更深層的籌碼訊號：

| 特徵名稱 | 說明 | 計算方式 |
|---------|------|---------|
| `fi_accel` | 外資買超加速度 | 近5日均值 vs 前5日均值 |
| `it_consecutive` | 投信連續買超天數 | 正值=連買N天，負值=連賣N天 |
| `chip_sync` | 籌碼共振 | 外資+投信同方向=1, 反向=-1, 其他=0 |
| `margin_price_diverge` | 融資股價背離 | 融資增+股價跌=-1, 融資減+股價漲=1 |
| `fi_net_ma5` | 外資淨買超5日均值 | 平滑化減少雜訊 |
| `it_net_ma5` | 投信淨買超5日均值 | 平滑化減少雜訊 |

- **影響範圍**：`features/feature_engineer.py`（CHIP_DERIVED 列表 + `_merge_chip_features` 方法）

### 2. 市場行情狀態辨識（4 個新特徵）

新增市場 regime 分類系統，讓模型「看得懂」現在是多頭、空頭還是盤整：

| 特徵名稱 | 說明 |
|---------|------|
| `market_regime` | 1=多頭, -1=空頭, 0=盤整（MA20 vs MA60 判定） |
| `regime_strength` | 趨勢強度（MA20 vs MA60 距離 / 股價，0~1） |
| `regime_duration` | 當前行情狀態持續天數 |
| `volatility_regime` | 波動率狀態（ATR 相對 60日的百分位，0~1） |

- **影響範圍**：`features/feature_engineer.py`（新增 REGIME_FEATURES 列表 + `_build_market_regime` 方法 + `_calc_regime_duration` 靜態方法）
- **特徵總數**：27 → 36 維

### 3. 行情感知信心度調整

預測結果會根據市場狀態自動調整信心度：
- 盤整行情（regime=0）+ 高信心 → 自動降級為中等信心（盤整時方向性弱，高信心通常不可靠）
- 高波動率（>0.8）→ 附加波動警告提示

- **影響範圍**：`workers/prediction_worker.py`

### 4. SHAP 中文標籤更新

新增 9 個特徵的中文解釋標籤，讓 SHAP 分析結果對使用者更友善。

- **影響範圍**：`models/lgbm_classifier.py`（`_feature_label` 方法）

### 5. TWSE API 全面修復（chip_fetcher.py 重寫）

#### 問題：TWSE API 格式變更
- **症狀**：籌碼資料完全無法抓取（0 筆），所有 API 回傳 404 或格式不符
- **根因**：TWSE 將 TWT38U（三大法人）和 TWT93U（融資融券）改版，從「指定個股+月份」改為「指定日期+全部個股」
- **修正**：
  - 三大法人改用 T86 日報 endpoint
  - 融資融券改用新版 TWT93U endpoint
  - 快取機制改為「每日一檔」（全部個股共用），大幅提升快取效率

#### 問題：`list index out of range`
- **症狀**：某些交易日（如 2025-12-24）T86 回傳的列欄位數不足 19 個
- **修正**：
  - 加入 `len(item) < 12` 跳過不完整的列
  - `item[18]`（三大法人合計）改為條件存取：不存在時自動用外資+投信+自營加總
  - TWT93U 也加 `len(item) < 13` 防護

#### 問題：TWSE 限流（307 重導向）
- **症狀**：連續打 API 後全部回傳 HTML 錯誤頁（「系統已達到使用量」）
- **修正**：
  - 使用 `requests.Session` 管理 cookie
  - 先訪問 TWSE 首頁取得 session cookie
  - `allow_redirects=False` 明確偵測 307/403/429 回應
  - API 間隔從 0.2s → 3s，限流等待從 5s → 15s
  - 連續失敗門檻從 5 → 3 次（更快偵測限流）
  - **從最近日期往回抓**（最新的籌碼最有價值，就算被限流也已拿到最重要的資料）
  - 被限流後仍繼續讀取已有的快取資料
  - 所有路徑（成功/假日/失敗）都加延遲
  - API 方法回傳 `(data, called_api)` tuple，精確區分快取命中 vs API 失敗

- **影響範圍**：`data/chip_fetcher.py`（完整重寫）

#### 效果
| 指標 | 修改前 | 修改後 |
|------|--------|--------|

| F1 Score | 0.0000 | 0.4552 |
| index out of range | 頻繁發生 | 已修復 |

5. **Release tag 格式必須是 `v1.x.x`** — updater 用 lstrip("v") 比對版本號

- **環境**：爸的電腦（Win10），從 v1.2.8 更新到 v1.3.0

- **需要檢查**：
  1. `auto_updater.py` 的 bat 腳本 xcopy 是否有覆蓋 `_internal/` 子目錄
  2. patch zip 裡是否包含 `_internal/version.json`
