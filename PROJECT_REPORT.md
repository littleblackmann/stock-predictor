## 一、系統概述

本系統為一套**桌面端台股漲跌預測工具**，結合深度學習（LSTM）、梯度提升樹（LightGBM）、自然語言處理（GPT）三大 AI 技術，對台灣股票市場進行次日漲跌機率預測。

系統特色：
- **多模型融合**：LSTM 時序特徵萃取 + LightGBM Ensemble 投票分類
- **多維資料來源**：技術面 + 籌碼面 + 美股隔夜 + 新聞情緒
- **自我進化**：增量學習 + 準確率驅動自動重訓
- **可解釋性**：SHAP 特徵重要度分析，每次預測附上驅動因子

## 四、模型架構

### 4.1 LSTM 時序特徵萃取器

**用途：** 從 60 天歷史價量序列中萃取時序模式，輸出 64 維特徵向量供下游分類使用。

```
輸入：60 天 × 17 維特徵矩陣
        ↓
LSTM(128, return_sequences=True) + Dropout(0.2)
        ↓
LSTM(64) → 64 維隱藏狀態（作為特徵輸出）
        ↓
Dense(1, sigmoid)（僅用於輔助訓練，推論時不使用）
```

| 參數 | 設定值 |
|------|--------|
| 回看窗口 | 60 個交易日 |
| 隱藏單元 | 128 → 64 |
| 訓練輪數 | 30（EarlyStopping patience=5） |
| 批次大小 | 32 |
| 優化器 | Adam（lr=0.001, ReduceLROnPlateau） |
| 正規化 | RobustScaler（抗離群值） |
| 資料切分 | 80 / 20，不打亂順序（保持時間序） |

> 每支股票獨立訓練並儲存：`models/saved/lstm_{SYMBOL}.keras`

### 4.2 LightGBM Ensemble 分類器

**用途：** 接收 LSTM 64 維時序特徵 + 技術 / 籌碼原始特徵，以 Ensemble 投票方式預測次日漲跌。

```
輸入：64 維 LSTM 特徵 + 13～20 維技術/籌碼特徵 = 77～84 維
        ↓
TimeSeriesSplit（3 折）
        ↓
┌─────────┐  ┌─────────┐  ┌─────────┐
│ Model 1 │  │ Model 2 │  │ Model 3 │
│ (Fold 1)│  │ (Fold 2)│  │ (Fold 3)│
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └───── 平均機率 ─────────┘
              ↓
         最終預測機率
```

| 參數 | 設定值 |
|------|--------|
| 提升類型 | GBDT |
| 棵數上限 | 500 |
| 學習率 | 0.05 |
| 葉節點數 | 31 |
| 最小分裂增益 | 0.01（自動過濾雜訊特徵） |
| 特徵子採樣 | 80% |
| 樣本子採樣 | 80%（每 5 輪） |
| L1 / L2 正則化 | 0.1 / 0.1 |

### 4.3 自我進化機制

系統具備三項自動進化能力，**越用越準**：

| 機制 | 說明 |
|------|------|
| **增量學習** | 重訓時以前代模型為 `init_model`，接續訓練而非從零開始（保留歷史知識） |
| **準確率驅動重訓** | 最近 20 筆預測準確率 < 52% → 自動觸發重訓（3 天冷卻期） |
| **定期重訓** | 模型超過 7 天未更新 → 自動重訓（吸收最新市場資料） |

### 4.4 信心評估

| 信心等級 | 條件 | 含義 |
|---------|------|------|
| 🟢 High | 距 50% ≥ 8% 且分散度 < 3% | 三模型高度一致，訊號明確 |
| 🟡 Medium | 距 50% ≥ 3% 且分散度 < 5% | 方向偏向明確，但非強力訊號 |
| 🟠 Low | 距 50% < 3% 或分散度 > 5% | 模型分歧大或接近隨機，建議觀望 |

## 五、新聞情緒分析與融合

### 5.1 新聞來源

```
Brave Search API（主要路徑）
    ↓ 無 API Key 或無結果時
Google News RSS（備用路徑）
    ↓
OpenAI GPT 分析
    ↓
情緒分數 + 影響評估
```

**Brave Search 多角度搜尋策略：**
1. 股票名稱 + 代號 + 「台股」
2. 所屬產業新聞
3. ETF 成分股相關新聞
4. 分析師文章

### 5.2 GPT 情緒分析輸出

| 欄位 | 範圍 | 說明 |
|------|------|------|
| `score` | -1.0 ~ +1.0 | 極度看空 ~ 極度看多 |
| `impact` | low / medium / high | 新聞對股價的影響程度 |
| `timeframe` | short / medium / long | 影響持續時間（1-2天 / 3-7天 / >7天） |
| `key_event` | 文字 | 最關鍵的單一事件摘要 |

### 5.3 情緒融合公式

為防止新聞標題的聳動性過度干擾 ML 預測，系統採用**雙重壓縮機制**：

```
1. 分數壓縮：compressed_score = 0.5 × tanh(1.0 × raw_score)
   → 原始 ±1.0 壓縮至 ±0.46，軟化極端值

2. 動態權重：weight = min(0.15 × impact_mult × timeframe_mult, 0.15)
   → 高影響 + 短期事件權重最高，但上限鎖定 15%

3. 融合：adjusted_prob = clip(raw_prob + compressed_score × weight, 0.05, 0.95)
```

| 影響程度 | 乘數 | 時間範圍 | 乘數 |
|---------|------|---------|------|
| low | 0.7× | short（1-2 天） | 1.2× |
| medium | 1.0× | medium（3-7 天） | 1.0× |
| high | 1.3× | long（>7 天） | 0.6× |

## 六、預測流程

完整預測一支股票的流程如下：

```
步驟 1 ─ 下載歷史資料
         yfinance 取得 1,500 天 OHLCV
              ↓
步驟 2 ─ 取得籌碼資料
         TWSE API 查詢三大法人 + 融資融券（快取 7 天）
              ↓
步驟 3 ─ 下載美股隔夜資料
         S&P 500、費半指數、VIX
              ↓
步驟 4 ─ 特徵工程
         技術指標 + 籌碼衍生 + 美股隔夜 + 週線 + 市場狀態
         輸出：20～27 維特徵矩陣
              ↓
步驟 5 ─ LSTM 訓練 / 載入
         60 天序列 → 64 維時序特徵向量
              ↓
步驟 6 ─ LightGBM Ensemble 訓練 / 載入
         LSTM 64 維 + 技術 13～20 維 → 3 模型交叉驗證
              ↓
步驟 7 ─ 新聞情緒分析
         Brave Search → GPT → 情緒分數 + 影響評估
              ↓
步驟 8 ─ 生成預測
         Ensemble 平均機率 → 信心評估 → 情緒融合
              ↓
步驟 9 ─ GPT 3 日走勢推估
         綜合技術面 + 情緒面 → 未來 3 天多空推估
```

## 七、可解釋性（SHAP）

每次預測皆附帶 **Top 5 驅動因子**，使用 SHAP TreeExplainer 計算每個特徵對預測結果的貢獻度：

```
範例輸出：
  • 週線 RSI 強弱：    看多 (+0.042)  ████████░░
  • 布林 %B 位置：     看多 (+0.031)  ██████░░░░
  • MACD 動能：       看空 (-0.025)  █████░░░░░
  • 外資連續買超：     看多 (+0.019)  ████░░░░░░
  • VIX 恐慌指數：     看空 (-0.012)  ██░░░░░░░░
```

- **正值（紅色）**：推動預測偏向「上漲」
- **負值（綠色）**：推動預測偏向「下跌」
- UI 以水平條形圖呈現，直覺易讀

## 十一、已知限制與風險聲明

### 技術限制
- 模型初期準確率約 52～55%，需持續使用 3～6 個月累積增量學習
- 籌碼面資料依賴 TWSE API，部分冷門股或 ETF 可能無資料
- 情緒分析品質受 GPT 模型版本與新聞品質影響
- 歷史回測不代表未來績效

### 投資風險警語

> ⚠️ **本系統僅供學術研究與參考使用，不構成任何投資建議。**
> 股票投資有賺有賠，過去績效不保證未來收益。
> 使用者應自行承擔所有投資決策之風險與後果。
